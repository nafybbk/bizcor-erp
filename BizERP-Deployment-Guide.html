Step 8 — Troubleshooting (Common Problems)

❌ Build fail ho rahi hai Vercel pe
Check karo:
  • Root Directory sahi set hai? Backend ke liye artifacts/api-server, Frontend ke liye artifacts/erp
  • Build Command exactly yeh hai? cd ../.. && pnpm install && pnpm --filter @workspace/api-server run build
  • Vercel → Project → Deployments → Latest deployment → "View Build Logs" mein exact error dekho
❌ "Invalid credentials" — Login nahi ho raha
Check karo:
  • Super admin ke liye Business Code field khaali rakhein
  • Bcrypt hash sahi hai? bcrypt-generator.com pe verify karo
  • Backend ka SESSION_SECRET env var set hai?
  • Backend URL VITE_API_URL mein correct hai?
❌ "Failed to fetch" ya API call fail ho rahi hai
Check karo:
  • Frontend ka VITE_API_URL exactly backend URL hai (no trailing slash)?
  • Backend deploy ho chuki hai aur running hai? https://bizerp-api.vercel.app/api/auth/me kholo browser mein
  • CORS: Backend mein CORS_ORIGIN frontend URL set hai?
  • Browser console (F12 → Console) mein exact error dekho
❌ Database connection error — "SSL required"
Fix: Backend environment variables mein DB_SSL=true add karo. Phir redeploy karo.
Verify: DATABASE_URL mein supabase.com domain hai — SSL automatic detect hota hai.
❌ Page refresh pe 404 error aata hai
Fix: Frontend project mein artifacts/erp/vercel.json file present honi chahiye jo pehle se code mein add kar di gayi hai. Agar nahi hai to create karo:
{"rewrites": [{"source": "/((?!assets/).*)", "destination": "/index.html"}]}
Phir git push karo → auto redeploy hoga.
❌ Supabase migration error — "type already exists"
Fix: Migration pehle se partially run ho chuki hai. SQL Editor mein yeh run karo pehle:
DROP TABLE IF EXISTS voucher_items, payment_allocations, payments, vouchers, items, custom_fields, hsn_codes, tax_rates, units, parties, users, app_settings, businesses, plans, super_admins CASCADE;
DROP TYPE IF EXISTS balance_type, billing_cycle, business_status, custom_field_entity, custom_field_type, discount_type, item_type, party_type, payment_mode, payment_type, user_role, voucher_status, voucher_type CASCADE;
Phir dobara migration SQL run karo.
❌ "Function timeout" error — Vercel serverless
Fix: Backend vercel.json mein maxDuration badha sakte ho. artifacts/api-server/vercel.json mein:
{"functions": {"src/vercel-entry.ts": {"maxDuration": 30}}}
