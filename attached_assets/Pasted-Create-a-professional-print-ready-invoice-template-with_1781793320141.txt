Create a professional print-ready invoice template with dynamic pagination.

Requirements:

1. Page Layout

- Top margin: 0.75 inch
- Bottom margin: 1 inch
- Left margin: 0.5 inch
- Right margin: 0.5 inch
- These margins must remain fixed on every page.
- Invoice must be optimized for A4 paper printing.

2. Dynamic Sections

- Header, company details, customer details, transport details, notes, terms, footer, etc. can grow or shrink depending on content.
- When any section grows (extra lines, longer text, additional information), it must consume space from the Line Items area only.
- The overall page margins must never change.

3. Line Items Area

- The Line Items table must be the only flexible section.
- It should automatically expand when there is more free space.
- It should automatically shrink when other sections become taller.
- The table should always fill the available space between the fixed top sections and fixed bottom sections.

4. Pagination Rules

- Never cut a line item row across pages.
- If the next row does not fully fit in the remaining space, move the entire row to the next page.
- Continue remaining line items on the next page automatically.
- Repeat table headers on every new page.
- Preserve page margins on all pages.

5. Footer Behavior

- Totals, tax summary, transport information, notes, signatures and footer content should stay together.
- Footer content must not overlap line items.
- If footer content cannot fit on the current page, move it to the next page.

6. Space Utilization

- Use all available space efficiently.
- If only a few items exist, the Line Items area should stretch and occupy the remaining space.
- If additional content is added elsewhere in the invoice, reduce the Line Items area accordingly.
- Avoid large blank gaps.

7. Technical Requirement

- Use CSS print layout suitable for PDF generation and browser printing.
- Use proper page-break handling.
- Support unlimited line items with automatic multi-page invoices.
- The generated PDF must look identical when printed.

Goal:
The invoice should behave like professional ERP invoices where margins remain fixed, content sections are dynamic, line items occupy the remaining available space, and additional pages are created automatically when required.
